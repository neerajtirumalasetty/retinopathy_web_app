# Diabetic Retinopathy Detection System

A full-stack, end-to-end web application that uses a trained MobileNetV2
Convolutional Neural Network to detect and classify diabetic retinopathy
severity from retinal fundus images.

This is the working implementation of the SDP project:
**"Diabetic Retinopathy Detection System – Early Disease Detection"**
(VIT-AP University, May 2026)

---

## 1. What this app does

1. User uploads a retinal fundus image and enters patient details
   (age, gender, diabetic history, symptoms).
2. The FastAPI backend loads the trained `best_mobilenetv2_gaussian.h5`
   model and runs **real inference** (no mocked/fake predictions).
3. The backend returns:
   - Predicted class (`No_DR`, `Mild`, `Moderate`, `Severe`, `Proliferative_DR`)
   - Confidence score
   - Risk level (Low / Medium / High / Critical)
   - Rule-based medical explanation
   - Recommended action
   - Per-class probability breakdown
4. The React + Tailwind frontend displays everything in a clean medical
   dashboard layout.
5. **Light/Dark theme toggle** — persists across sessions (localStorage),
   defaults to system preference on first visit.
6. **Language selector** — supports English, Hindi, and Telugu. All UI
   text (labels, buttons, results, risk levels, class names) is translated.
   Persists across sessions. Adding a new language only requires adding
   one new object to `frontend/src/i18n/translations.js`.

## 3. Backend Setup (FastAPI)

### Requirements
- Python 3.10+
- pip

### Install & Run

```bash
cd backend

# (recommended) create a virtual environment
python3 -m venv venv
source venv/bin/activate        # on Windows: venv\Scripts\activate

# install dependencies
pip install -r requirements.txt

# make sure your trained model is at:
# backend/models/best_mobilenetv2_gaussian.h5

# run the server
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

The API will be available at: **http://localhost:8000**

### Verify it's working

```bash
curl http://localhost:8000/health
# {"status":"ok","model_status":"loaded"}
```

### API Docs

FastAPI auto-generates interactive docs at:
**http://localhost:8000/docs**

## 5. API Reference

### `POST /api/predict`

**Content-Type:** `multipart/form-data`

| Field              | Type   | Required | Description                                              |
|--------------------|--------|----------|------------------------------------------------------------|
| `file`             | image  | Yes      | Retinal fundus image (jpg/png)                            |
| `age`              | int    | No       | Patient age                                                |
| `gender`           | string | No       | Patient gender                                             |
| `diabetic_history` | string | No       | Years with diabetes                                         |
| `symptoms`         | string | No       | Free-text symptoms                                         |
| `language`         | string | No       | `"en"` (default), `"hi"`, or `"te"` — language for the returned recommendation, medical_explanation, and safety_note |

> **Note:** the `language` field controls the *content* generated by the
> backend (recommendation / explanation / safety note). The frontend's
> theme & language selector (top toolbar) automatically sends the
> currently selected language with every request — UI labels and backend
> content stay in sync without any extra wiring.

**Response:**

```json
{
  "prediction": "Moderate",
  "confidence": 93.24,
  "risk": "Medium",
  "recommendation": "Schedule an ophthalmology consultation within the next few weeks for a detailed evaluation.",
  "medical_explanation": "Moderate diabetic retinopathy detected. ...",
  "safety_note": "Watch for blurred vision, floaters, or dark spots and report them promptly.",
  "class_probabilities": {
    "No_DR": 0.26,
    "Mild": 1.47,
    "Moderate": 93.24,
    "Severe": 1.64,
    "Proliferative_DR": 3.39
  }
}
```

## 6. Model Details

- **Architecture:** MobileNetV2 (transfer learning, ImageNet weights) +
  GlobalAveragePooling2D + Dropout(0.3) + Dense(5, softmax)
- **Input size:** 224×224×3
- **Preprocessing:** RGB conversion, resize to 224×224, normalize to [0,1]
  (`pixel / 255.0`) — matches the training pipeline in the SDP notebook.
- **Classes:** `No_DR`, `Mild`, `Moderate`, `Severe`, `Proliferative_DR`
- **Reported validation accuracy:** ~73.8% (per SDP Final Report, Chapter 4)

| No_DR             | Low      |
| Mild              | Low      |
| Moderate          | Medium   |

## 7. Notes on the Medical Explanation Module

The SDP report describes an LLM (Gemma-2B-IT) for generating medical
explanations. For this production deployment, a **deterministic
rule-based explanation engine** (`backend/explanation.py`) is used
instead. This is intentional:

- It requires no external API key or GPU for the language model.
- It is 100% reproducible and safe for a demo/viva.
- It still incorporates patient context (age, diabetic history, symptoms)
  into the generated explanation, satisfying the same functional
  requirement described in the report.

If you want to swap in a real LLM later, simply replace the function
bodies in `explanation.py` with calls to your LLM of choice — the
FastAPI endpoint signature does not need to change.

## 8. Theme & Language Customization

### Adding a new language
This app has **two** translation layers that both need updating for a new
language to be fully supported end-to-end:

**1. Frontend UI labels** — `frontend/src/i18n/translations.js`
Add a new key, e.g. for Tamil:

```js
ta: {
  appTitle: "...",
  // ...copy every key from the `en` block and translate
},
```

Then register it in `frontend/src/context/LanguageContext.jsx`:

```js
export const LANGUAGES = [
  { code: "en", label: "English" },
  { code: "hi", label: "हिन्दी (Hindi)" },
  { code: "te", label: "తెలుగు (Telugu)" },
  { code: "ta", label: "தமிழ் (Tamil)" }, // new
];
```

**2. Backend generated content** — `backend/explanation.py`
The recommendation, medical explanation, and safety note text are
generated server-side (not in the frontend dictionary), so add the new
language code to `SUPPORTED_LANGUAGES`, then add a `"ta": {...}` entry to
each of `BASE_EXPLANATIONS`, `RECOMMENDATIONS`, `SAFETY_NOTES`, and the
`_CONFIDENCE_TEMPLATE` / `_HISTORY_TEMPLATE` / `_SYMPTOM_TEMPLATE` /
fallback dictionaries, mirroring the structure already used for `en`/`hi`/`te`.

Once both layers have the new language code, it will appear in the
dropdown and the backend will return fully localized predictions for it —
no other code changes needed.

### Customizing the theme
Dark mode uses Tailwind's `class` strategy (`darkMode: "class"` in
`tailwind.config.js`). To adjust dark-mode colors, edit the `dark:` variant
classes directly in the component files, or extend the `medical` color
palette in `tailwind.config.js` to add more dark-mode-friendly shades.

## 9. Troubleshooting

| Issue                                   | Fix                                                                 |
|------------------------------------------|----------------------------------------------------------------------|
| `Model file not found`                   | Ensure `best_mobilenetv2_gaussian.h5` is inside `backend/models/`   |
| CORS errors in browser console           | Confirm backend is running and `CORSMiddleware` allows your origin  |
| Frontend can't reach backend             | Check `API_BASE_URL` in `frontend/src/api.js`                       |
| Slow first prediction                    | Normal — model is loaded into memory on server startup              |
| `ModuleNotFoundError` on backend startup | Run `pip install -r requirements.txt` inside your virtual environment |
